##  交互准则 (Interaction Rules)
1.命名预审：在生成任何新函数、宏定义或结构体之前，**必须先列出建议名称方案**，询问我的意见并获得许可后再编写具体代码。
2.代码解释：禁止在代码注释中进行长篇大论的各种逻辑解释。就是向我解释的部分，注释就是注释本身，不要在注释里放给我的解释。
3.上下文窗口：每次压缩上下文之后会遗忘这个规范文档的内容，提醒我再发一次。
4.变量修饰解释：凡是代码中使用了 `const`、`static`、`volatile` 等修饰符，必须在代码块外解释其使用原因。
5.外部接口化：鼓励通过外部接口（API）调用的方式解耦逻辑，不要在模块内部直接操作其他模块的私有成员。
6.原子操作意识：在 main loop 和中断同时访问同一个全局变量（如传感器数据）时，必须提醒我使用临界区保护（如 __disable_irq()或__set_PRIMASK(1)）。

##  命名规范 (Naming Convention)
1.宏定义：全大写，单词间用下划线分隔。例如：`MAX_PWM_VALUE`。
2.变量命名：
    开头必须小写
    拒绝过度缩写：除非是极通用的（如 `i`, `j`），否则严禁简写。例如：使用 `txBuffer` 而非 `txBuf`。
    风格严谨统一：严禁在一个项目中混合使用 `dmaTxBuf`  和 `frame_buffer` 这种明明是同一单词但是一个大写一个小写的情况，请在生成前确认当前项目的统一风格。
    较短的变量名不准用下划线，较长的变量名要求使用下划线.
3.函数命名：大小写、下划线均可使用，但需遵循逻辑语义（如 `Motor_Move_To_Position`）。
4.数字：严禁魔法数字，使用define的全放在头文件，根据类型不同用换行隔开
5.特殊要求：
    1）receive和transmit可以缩写成rx和tx，position可以缩写成pos这样的。

##  代码结构与注释 (Structure & Comments)
1.文件格式：分离 `.c` 和 `.h` 文件。
2.头文件包含：所有 `#include` 统一在 `.h` 头文件中进行。
3.注释风格：全中文注释。必须简洁到位。不要在代码块中放长篇大论的解释。不是不要在代码块中写注释的意思。不要在注释中放你对一次对话中提出的更改的解释，即沟通性释义，在贴出代码前或后进行解释。宏定义必须加注释。函数开头（仅限函数！）必须使用以下格式的注释：
    /**
     * @brief 
     * @param 
     * @retval 
     */
     但是严禁在头文件里使用这种注释。
4.作用域控制：严格遵循“不污染”原则。多使用结构体封装、`static` 私有化变量，通过函数接口进行数据交互，静态变量不一定要封装成结构体，那样反而臃肿，但凡是结构体封装统一放在头文件里。
5.句柄传递原则：编写外设初始化或控制函数时，优先传递 HAL 的句柄指针（如 UART_HandleTypeDef *huart），而不是在函数内部写死&huart1。
6.代码格式要求：
    1）函数的花括号、循环的花括号、逻辑判断的花括号等，均使用示例的格式。
        示例：
            if ()
            {

            }
            或者
            void function(void)
            {

            }
    2）关联度小的代码块一定要用换行隔开一行。
    3）如果是较简短的逻辑，可以单行写完的if判断之类的，就不要用花括号。

##  外设与逻辑开发准则 (Development Rules)
1.非阻塞核心：严禁使用任何形式的硬件阻塞等待（如 `HAL_Delay()` 或死循环等待标志位）。必须使用状态机、定时器计数或标志位轮询。除非是刚上电初始化那会，或是不得不用阻塞等待。
2.串口通讯：统一采用 **DMA + IDLE（空闲中断）+ FIFO（环形缓冲区）** 架构。
3.中断逻辑：中断服务函数 (ISR) 必须保持极简，严禁在中断内进行复杂算法计算、延时或打印。
4.安全保护：所有的控制输出（PWM、速度、角度）必须在底层驱动或逻辑层做好限幅和安全保护逻辑。
5.参数校验：所有通过指针传递的外部接口，在函数开头增加 if (handle == NULL) return; 这样的防错处理，以增强强鲁棒性。
6.固有配置：
    1）将debug刷新标志位放到定时中断里进行刷新。
    2）灰度传感器数据、编码器数据的读取在定时中断里进行。
    3）按钮的代码可以使用阻塞消抖，因为是担心阻塞影响自动控制，但按钮在使用的时候一般是在人为手动调教。
    4）debug的屏幕空间很充足，尽量不使用缩写，同时行间隔为20.
    5）
